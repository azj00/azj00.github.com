    Our Goal: Recreate an English learning mobile app which allows users to have realistic conversations in English with an AI bot from scratch.

    The Added Feature

    In the feature ideation meeting, I suggested that we can add a Short Exercise card in the front page.

    Why short exercise?
  • Allow users to practice just one tap away.
  • Make the AI bot more like users' friend.
  • Get to know users' interests: for personal recommendation.

    • Submitted Version

    My design after the project is due

    But asking the same question or a question out of nowhere will probably be ignored? How can we get users' attention?

    We can use the AI to create questions that related to the users' daily lives or personal interests. In addition, based on users' response rate, we can improve the AI's response to make it "understands" users.

    The algorithm:
  • If the user is new to the app, the default question is "How are you today?"
  • If the user mentioned a future event in the chat before, ask the outcome of the event.
  • Else if the user only use the app for several days && don't have a terminated English level, ask questions that contains current level words and grammar.
  • Else if the user mentioned a specific object/event for more than 5 times in a chat, ask questions about that object
  • Else if the user has a strong preference on a topic, ask random questions related to that topic
  • Else: ask a question based on the last practiced topic
  • The random question is generated by the news and answers from other users who practiced the same topic
  • If the user answer the question in a topic, the personal recommendation algorithm will increase the ranking of this topic and learn the keywords for the next question
    • The startup likes our idea and will implement this feature soon!

    The Deleted Feature

    When playing around the sketch, I found that the "self assessment" feature is unnecessary.

    As a user
  • Bothersome to give out personal information to a “stranger” app
  • Too long and time consuming
  • Too much pressure
    • As a designer
  • A long questionnaire is necessary to correctly justify users' English level.
  • Spoken English cannot be assessed by school performance

    • As a student who studies Computer Science, I believe that it is possible to let AI evaluate users’ English proficiency during the chat. I created a rough algorithm for the implementation.

    English Proficiency Test Algorithm In a Chat
  • Rank words and grammar rules in different levels
  • Start the conversation with the lowest level and gradually increase the level of difficulty
  • Assess a user's vocab level by the words users miss
  • Assess a user's grammar level by the grammar mistakes
  • If the user missed 20%* of the vocab/grammar in a level, the AI will continue to the next level
  • If the user missed 50% of the vocab/grammar in a level, the AI will continue the chat mixing next level's words/grammar
  • If the user missed 80% of the vocab/grammar in a level, the AI will set the level as teaching goal
  • Train and improve a model based on algorithm
    • *:The percentage of missed rate should be adjusted based on experiment

    The group all agree with me that the self-assessment is unnecessary and believe the algorithm makes sense.

    Mockup Critics in Class

    After finishing the hi-fi prototype, we submitted our work to the class for the first round critique.

    Major Changes After the in-class Critique
    Critics Summary
    • Text across the app was too small. Specifically, “view all” buttons, “browse all”, topic names, and sign up instructions, which we made bigger.
    • The login screen flows were confusing. We streamlined this into one sign up page and one login page.
    • We had inconsistent use of bookmark icons for saved topics. We added bookmark icons to every topic to allow users to save topics.
    • We had inconsistent titles for the “Continue Practicing” sections on the home and notebook pages. We made these sections’ titles the same to make it clear these were the same section.
    • We used red text for mistake corrections, which was seen as too aggressive. We changed this to blue.
    • Our prototype did not show that corrections were made in real time on the chat screen. We added grammatical corrections and a pop-up explanation.
    • To better use the touch interface, we added a swipe up functionality to continue the conversation on the transcript screen.
    • The profile page had no way to edit account details, so we added edit buttons.

    Online User Testing

    To better understand our product's usability, we submitted our revised prototype to a remote user testing service (UserTesting.com) for real user testing.

    Tasks Summary
    1. Try to chat about the topic “Social Gathering”
    2. Continue the conversation about “Snack Time.”
    3. Go back to the transcript from chatting about "Snack Time"
    4. View all topics from home page
    5. Sign out the app
    Difficulties
    1. All succeed. One user suggest to bold the title.
    2. All struggled. The drag up functionality was not intuitive.
    3. All struggled. They were confused because of the failure of task 2.
    4. All found it easy.
    5. All found the button where they expected to see.
    Potential Interface Changes
    • Replace the dragging function with our original design: button. But we also think this difficulty is caused by the fact that Figma requires to drag to mimic the behavior of swipe up. Users won't find it difficult if they play the prototype on an actual device.
    • Add a "start a new conversation" option to users on a topic they practiced before.
    • Change the topic images more relevant to the topics.
